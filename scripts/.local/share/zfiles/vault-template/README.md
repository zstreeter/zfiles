# Research Vault

## Layout

- `literature/` — one note per Zotero item (auto-generated by Obsidian Zotero Integration)
- `notes/` — freeform research notes, link to `literature/` with `[[wikilinks]]`
- `drafts/` — `.qmd` drafts in progress; `quarto preview` works from this vault root
- `assets/` — images and attachments
- `references.bib` — Better BibTeX auto-export target (set this up in Zotero per-vault)
- `_quarto.yml` — local Quarto config; renders `drafts/**/*.qmd`

## First-time setup (per vault)

### 1. Obsidian community plugins
Open this folder in Obsidian, then **Settings → Community plugins → Browse** and install:

- **Zotero Integration** (mgmeyers) — pulls Zotero items into `literature/` with annotations
- **Citations** (hans) — `@citekey` autocomplete from `references.bib`
- **Dataview** — query notes by tag/field
- **Templater** — note templates

After install, enable each.

### 2. Better BibTeX auto-export (Zotero)
In Zotero: **Edit → Preferences → Better BibTeX → Automatic export**, target this vault's
`references.bib` with format **Better BibLaTeX**. Set "On change" so it stays current.

### 3. Publish a draft to the blog
```
publish-post drafts/my-post.qmd          # uses filename as slug
publish-post drafts/my-post.qmd my-slug  # explicit slug
```
This copies the draft + adjacent images into
`~/Documents/Repos/zstreeter.github.io/posts/YYYY-MM-DD-slug/index.qmd`.
Use a `categories:` field in the draft's frontmatter so the post groups under your project on the blog.
